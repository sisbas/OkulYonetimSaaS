# PR-KVKK-LOG-01 — CI Log Sensitive Pattern Scan

## Purpose

Add a post-quality-gate sensitive pattern scan for `ci-logs` artifacts. If the scan finds an unredacted sensitive pattern, Sprint 1 Quality Gate must fail.

## Scope

The scan covers log artifacts generated by `scripts/ci-quality-gate.sh`:

- credential / password / secret / API key style key-value output
- token / bearer token output
- email values
- phone values, including common Turkish mobile formats
- parent / guardian contact key-value output
- `message_body` key-value output
- `guidance_note` / `rehberlik_notu` key-value output

## Disclosure control

The scan does not print matched values to the GitHub Step Summary. It writes only file, line, and category to:

```text
ci-logs/sensitive-pattern-scan.tsv
```

## Failure behavior

Any finding creates the Sprint 1 Quality Gate summary row:

```text
11 sensitive log scan | FAIL | audit/redaction failure
```

The workflow exits non-zero before the final PASS decision.

## Non-goals

- No new product feature
- No change to RBAC/KVKK/audit tests
- No test weakening
- No TypeScript/lint suppression
